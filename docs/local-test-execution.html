4 · Per-test lifecycle (repeated for each test in the run)
▶︎   A Test Runs  ·  opens a fresh browser context loaded with storageState
Hooks · before
beforeAll  →  beforeEach   — set up fixtures (none required; storageState carries the session)
Open context
new browser context with use.storageState   — the page starts already logged in, no per-test login
test() body
setJiraStory('PROJ-1') — annotate the parent user story (first line of every test)
then: test.step('Open Desk', …)  →  page object  →  ActionKeyword  →  Playwright  →  the SUT
Assertions
expect(...)   /   expect.soft(...) chains   — whole chain validated in one run (soft); hard expect stops on first miss
On failure
friendly error "Step failed: could not click ..."  +  screenshot, trace, video  →  retry × N (N = test.retries, default 3 per spec)
afterEach
fixture
jiraBugOnFailure auto-fixture (helper/test.ts) inspects the result. On pass → return. On fail, branches by attempt:
⏳ Non-final attempt
retry < maxRetries → show a deferred step "Will retry (attempt N/M) — bug report deferred". No Jira call yet.
🐞 Final attempt failed
retry === maxRetries → read setJiraStory annotation → search Jira: existing OPEN bug? reuse ✅ — else create + link to story ✅. Bug key shown as a nested step.
Hooks · after
afterEach  →  afterAll  — context closed; storageState file is untouched (read-only for tests)
