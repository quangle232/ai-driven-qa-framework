Framework Capabilities
MyApp Automated Testing  ·  Playwright + TypeScript  ·  Page Object Model  ·  Jenkins CI
🔐
Login Once · Reuse Everywhere
Sign in ONCE per build in global-setup, save storageState to disk. Every test & every worker reuses it — no per-test login, no daily session-limit spam on the SUT. refresh=yes only when needed; warms Desk + CRM in one pass.
helper/authenticate-set-up.ts
📦
Page Object Model
One class per screen, all extend BasePage. Locators grouped in fields / locators; methods are page actions. A UI change is fixed in exactly one class.
page-objects/
⚙️
Single Keyword Layer
ActionKeyword is the only layer that calls Playwright. Self-healing locators (data-zcqa → data-test-id), iframe + reload-poll for async the SUT values, async-safe getters.
helper/action-keywords.ts
🪶
Stakeholder-Friendly Errors
Playwright timeouts → plain English: “Step failed: could not click option "Demolition Request" — not visible within 15s.” Raw selector kept on a second line as a debug hint.
friendlyLabel / friendlyWait
⚡
Parallel Execution
Playwright --workers 1–10 (Jenkins choice) runs tests concurrently across browser contexts. All workers share the same storageState → already-logged-in, zero extra logins even with 10 parallel browsers. Build time drops 5–10×.
WORKERS · config/playwright.config.ts
👻
Headless by Default
On CI, headless = default (CI=true) — faster, no display server needed, perfect for parallel workers. Local / Jenkins HEADED param flips to headed when debugging. global-setup stays headed so 2FA / OTP can be done manually if the SUT prompts.
HEADED · CI · use.headless
🌐
Multi-environment Runs
cross-env test_env=sandbox|uat + dotenv load per-env URLs, credentials, sandbox IDs. Same suite runs against any environment without code changes.
environments/.env.<env>
🎞️
Trace · Video · Screenshot
On failure Playwright auto-retains trace.zip + video.webm + screenshot.png. Open the trace in Playwright Inspector → step-by-step time-travel of the failing run.
retain-on-failure · test-results/
🐞
Auto Bug on Failure
On failure: search the project for an OPEN bug with the same summary. Reuse if found; otherwise create a new Bug and link it to the parent user story declared by setJiraStory(). No spam, no orphan bugs.
helper/jira-bug-reporter.ts
🏗️
Jenkins CI Pipeline
Parameterized declarative pipeline: BRANCH, TAGS, ENV, INVERT_BUGS, REFRESH_STORAGE, RETRIES, WORKERS, HEADED. Allure + artifact archive built-in.
jenkins/web-regression-job
🏷️
Trigger by Tag  ·  PR / CI / manual
trigger-jenkins.js <tag> fires the regression job from any source — a dev PR pipeline, the terminal, scheduled cron. --no-wait + --status for long builds (no blocking poll loops). Tag == Jira label.
trigger-jenkins.js
📧
Email + Allure Report
Per-build HTML e-mail to stakeholders: pass/fail summary, project matrix, failed-tests table, links to the Jenkins build + the per-build Allure report.
jenkins/scripts/collect-playwright-stats.js
Every capability is driven by the same tag the user story carries in Jira — tag == Jira label — so a single value links the test, the run, the bug, and the report.
